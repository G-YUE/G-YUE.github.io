swapon新增

发表于 2017-03-19

swapon -s 查看当前swapon信息
新增swap分区步骤：

mkswap /dev/sdb8

swapon -a /dev/sdb8

卸载swap分区。

swapoff /dev/sdb8

free 查看内存信息

[root@linuxso.com ~]#free -m
        total    used   free   shared    buffers   cached
   Mem: 503      368     134        0      56          249
-/+ buffers/cache: 62    440
   Swap: 1023     0     1023

Mem：表示物理内存统计

-/+ buffers/cached：表示物理内存的缓存统计

Swap：表示硬盘上交换分区的使用情况，这里我们不去关心。

第1行 Mem：

total：表示物理内存总量。

used：表示总计分配给缓存（包含buffers 与cache ）使用的数量，但其中可能部分缓存并未实际使用。

free：未被分配的内存。

shared：共享内存，一般系统不会用到，这里也不讨论。

buffers：系统分配但未被使用的buffers 数量。

cached：系统分配但未被使用的cache 数量。

available: 可以使用的内存总量

buffer 与cache 的区别见后面。

total = used + free

第2行 -/+ buffers/cached：

used：也就是第一行中的used - buffers-cached 也是实际使用的内存总量。

free：未被使用的buffers 与cache 和未被分配的内存之和，这就是系统当前实际可用内存。

free 2= buffers1 + cached1 + free1 //free2为第二行、buffers1等为第一行
buffer 与cache 的区别：

buffer是即将要被写入磁盘的，而cache是被从磁盘中读出来的

清楚cache缓存：

echo 3 > /proc/sys/vm/drop_caches

uname -a 查看当前系统的所有信息。有内核版本、CPU

cat /etc/redhat-release 查看当前操作系统的版本。

进程线程

发表于 2017-03-17 | 分类于 linux学习基础

进程的查看。

ps aux 查看当前系统的所有进程

ps aux |grep  名字  grep；过滤作用

将系统进程过滤，只展示有上述名字的进程。

pgrep 只查询进程的PID号

pgrep 想要查看的名字

只显示 想要查看的名字的PID号

firefox 打开火狐浏览器。会占用当前终端

firefox &
不占用当前终端，后台打开火狐浏览器。

jobs 查看当前终端后台运行的程序的信息。有工作好。

#jobs
[1]+ 运行中     firefox &


杀死进程
#kill -9 %1

#jobs
[1] 已杀死    firefox

进程的杀死。

kill 根据进程PID号或者进程名

kill -9 PID号  强制杀死。

pkill -9 进程名

pstree 查看进程树

-p    显示所有的时程，显示结果包含进程名和时进程ID

-u    显示出用户的UID,无论何时，这个UID和进程比较UID参数，这个新的UID将在进程名后显示不同的参数。

-U 　使用utf-8字符集以十进制表示,

-v     显示版本号

dd if=/dev/zero of=/a.txt bs=1G count=1

从zero中复制1个1G到a.txt中：

if 代表输入文件。如果不指定if，默认就会从zero中读取输入。

of 代表输出文件。如果不指定of，默认就会将a.txt作为默认输出。 

bs 将输入与输出设成指定的字节数。 

count 仅读取指定的区块数。

可以测试硬盘的读写速度。

软件包的安装和管理

发表于 2017-03-17 | 分类于 linux学习基础

软件包在linux里有两种:

二进制形式的包（rpm包）：软件包名-版本号3位-修正次数-应用平台-软件包格式。

源代码包：脚本安装包

安装二进制的包：

本地包都在 /run/media/root/Packages中

rpm -ivh vsftp-3.0.2-9.el7.x86_64.rpm 安装vsftp软件包

rpm -qi vsftpd 显示软件包的详细信息

rpm -ql vsftpd 显示安装后的位置详细信息。

rpm -qa 查看系统中安装了的所有包。

rpm -qa |grep vsft 大致名

或者

rpm -q vsftpd  全名

rpm -qf 路径： 查看输入的路径是由哪个包形成的。

查看一个命令对应文件的真实位置。
which 命令 返回命令对应文件的路径

例如：which ls   会返回存储ls真实源代码的路径

卸载二进制软件包

rpm -e 软件包名字

rpm安装包弱点：如果碰上需要安装的包的依赖包没装，会提示失败，需要安装依赖包后再安装需要的包。这就很麻烦了。所以有一个yum命令来安装包，yum会自动安装所有的依赖包。

yum安装二进制包。

yum install 包名。

会自动检测依赖关系，并下载依赖包。安装时提示是否安装。如果不想要提示，可以：

yum -y install 包名

安装yum组包：

yum grouplist   会显示组包列表
yum groupinstall 组包名

重新安装:

yum reinstall 包名

清楚yum缓存：

yun  clean all

yum卸载：

yum -y rmove（-e erase） 包名

给yum创建缓存，提高yum安装速度。

yum makecache -y

自己创建一个本地yum仓库。

将以后安装的包保存到本地一个指定文件夹中：

打开 /etc/yum.conf文件，更改内容如下：


创建一个仓库目录。

下载包到仓库中。

更改 /etc/yum.repos.d/下的repo文件。全部剪切到别处，新建yue.repo。名字随便。然后写入：    
[yue]
name=gaoyue
baseurl=file:///createrepo  #本地仓库地址
enabled=1
gpgcheck=0

createrepo 仓库路径。 在仓库中创建一个依赖关系的目录。

安装源码包：以python3示例。python-3.6.0.tgz

在官网下载源码包，
解压包 tar -xvf python-3.6.0.tgz

python是用C开发的，需要安装开发工具 gcc glibc。

yum grouplist   会显示组包列表

yum groupinstall 开发工具

定义安装后的路径：

cd  解压后的python3.6目录中

./configure --prefix=/usr/local/python3.6    #设置安装路径

安装：

make
make install

配置环境变量：

cd /etc

vim profile 打开文档

export PATH=/usr/local/python3.6/bin:$PATH 在文档最下面新增这句保存推出。

重启系统，就可以在命令行输入python3.6，进入python shell环境了。
