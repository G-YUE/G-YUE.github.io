linux硬盘分区、格式化及系统的管理

发表于 2017-03-16 | 分类于 linux学习基础

首先看下各种存储设备在Linux中的文件名：

为什么进行分区？

  • 更容易管理和控制系统。
  • 系统效率更高。
  • 可以限制用户使用硬盘的空间大小。
  • 更容易备份和回复。

Linux系统中硬盘的分区

硬盘的分区可以分为：主分区、扩展分区和逻辑分区。

一块硬盘上最多可以划分出4个主分区。

df: 查看磁盘分区，大小，挂载等内容。
参数：
    H: 以1024模式显示    
    i:显示inode信息，有inode码之类的。

下面来用fdisk来创建磁盘分区：

fdisk：创建磁盘分区。

首先，fdisk 要被分区的磁盘名。

fdisk命令列表中常用的命令。
d: 删除一个（已经存在的）分区，delet

l: 列出（已经存在的）分区的类型，list

m： 列出fdisk中使用的所有命令。

n： 添加一个新的分区，new。

p： 列出分区表的内容，print。

q： 推出fdisk，但是不会存储所做的变化，quit。

t： 改变分区系统的id，title。

w： 推出fdisk并储存所做的变化，write

创建完分区还不能立刻是用，需要对分区进行格式化。

mkfs：创建文件系统，格式化分区。

首先了解下常用的文件系统类型。

ext2：Linux系统中标准的文件系统
ext3：一种日志式文件系统
ext4：一种针对ext3系统的扩展日志式文件系统
lvm：逻辑盘卷管理 
iso9660：目前唯一通用的光盘文件系统 
XFS： 一种高性能的日志文件系统

例如：要将/dev/sdb1 格式化为xfs文件系统。

mkfs.xfs /dev/sdb1

mount：挂载。

挂载：指将一个设备挂机到一个已存在的目录上。

例如：将/dev/sdb1 分区挂载到/wg目录上。

mount /dev/sdb1 /wg

卸载：将目录上的挂载卸载掉。

例如：卸载/wg上的文件系统。

umount /wg

软链接和硬链接。

软链接：ln -s 源文件路径 快捷方式所在路径

快捷方式指向的是源文件的文件名，有自己的innode。删除源文件，快捷方式失效。可以跨分区连接。

硬链接：ln 源文件路径 快捷方式所在路径

快捷方式指向的是源文件的innode号。删除源文件，快捷方式不受影响。不可以跨分区链接。
