网络配置、ssh、scp

发表于 2017-03-21 | 分类于 linux学习基础

ifconfig

eno33:网卡名

mtu 1500:最大发送字节为1500

ether：mac地址

更改网络地址：

ifconfig eno33 192.168.16.123/24

配置网关：

route add default gw 192.168.16.254 netmask 255.255.255.0

删除

route del default gw 192.168.16.254 netmask 255.255.255.0

永久修改：

cd /etc/sysconfig/network-scripts 

该目录下有一个ifcfg-网卡名的配置文件


HWADDR=''    #网卡mac地址
IPADDR=192.168.16.123   #IP地址
NETMASK=255.255.255.0   #子网掩码
GATEWAY=192.168.16.254  #网关地址
DNS1=192.168.16.254  #DNS服务器地址

重启网络服务：

systemctl restart network

本地解析域名文件：/etc/hosts文件

ip地址    域名

访问指定域名，把域名解析成指定的IP地址。

修改主机名：

honstname  #查看主机名

hostnamectl set-hostname 主机名  #更改主机名

远程连接：两台linux系统计算机：前提sshd服务启动

主机A ： ssh 192.168.16.123    #主机B的IP地址

然后确认，填写ROOT密码就可以登录了。

远程拷贝:

scp /etc/hosts  192.168.16.123:/tmp #把主机A的host文件拷贝到192.168.16.123下的/tmp下 

如果hosts文件中对192.168.16.123设置了域名解析。可以把IP地址换成域名，也可以完成远程拷贝。

2个主机位置可以互换，就变成了远程下载。

ssh-keygen:直接默认 制作钥匙

#ls /root/.ssh/

id_ras#钥匙  id_rsa.pub#锁

将锁发送给服务端：

ssh-copy-id -i  （文件名） 服务端

然后客户端连接服务端就不需要密码了。

ssh配置文件：

/etc/ssh/sshd_config    可以更改端口。

ssh 192.168.16.123 -p 更改后的端口号
